## 개인 R&D 엔지니어링 허브 앱
### 버전: v0.1 MVP 초안
### 작성일: 2026-03-09
### 대상 사용자: 박현수(1인 개발/운영 기준)
---
# 1. 제품 개요
## 1.1 제품명(가칭)
* **LabPilot**
* 부제: **개인 R&D 프로젝트 운영 허브**
## 1.2 제품 한 줄 설명
임베디드, 하드웨어, 펌웨어, 실험, GitHub/GitLab 이슈를 한곳에서 관리하는 **개인 연구개발 운영 앱**
## 1.3 제품 비전
흩어진 개발 기록, 실험 로그, 하드웨어 이슈, 코드 저장소 이슈를 통합하여
**“무엇을 만들고 있는지, 지금 어디까지 왔는지, 다음에 무엇을 해야 하는지”**를 한눈에 파악할 수 있는 엔지니어링 전용 허브를 제공한다.
---
# 2. 문제 정의
현재 사용자는 다음과 같은 문제를 겪고 있다.
## 2.1 프로젝트 정보가 여러 곳에 흩어짐
* GitHub와 GitLab이 동시에 존재
* 프로젝트 문서, 이슈, 커밋, 실험 결과, 하드웨어 작업 기록이 분산됨
* 특정 문제의 원인과 해결 이력을 다시 찾는 데 시간이 오래 걸림
## 2.2 하드웨어/실험 작업은 일반 PM 툴로 관리가 어려움
* 일반 이슈 트래커는 실험 조건, 측정 결과, 사진, 배선 상태 등 하드웨어 문맥을 충분히 담기 어려움
* “센서 교체 후 문제 발생”, “특정 커밋 직후 LoRa 이상” 같은 인과관계 추적이 어려움
## 2.3 포트폴리오/보고서 작성 비용이 큼
* 프로젝트 수행 중에는 기록이 남아도
* 나중에 발표자료, 자소서, 포트폴리오로 다시 정리하는 비용이 큼
## 2.4 현재 상태 파악이 어려움
* 프로젝트가 여러 갈래로 진행됨
* 지금 막혀 있는 지점, 다음 액션, 치명 이슈를 빠르게 보기 어려움
---
# 3. 목표
## 3.1 제품 목표
이 앱은 다음 4가지를 해결해야 한다.
1. **프로젝트 진행상황 통합**
2. **실험 로그와 하드웨어 이슈의 구조화**
3. **GitHub/GitLab 활동의 통합 조회**
4. **향후 포트폴리오/리포트 자동화의 기반 마련**
## 3.2 MVP 목표
MVP 단계에서는 아래를 달성하면 성공으로 본다.
* 프로젝트를 생성하고 관리할 수 있다
* 실험 로그를 작성하고 조회할 수 있다
* 하드웨어 이슈를 등록하고 추적할 수 있다
* GitHub/GitLab 저장소를 연결하고 최근 이슈/커밋을 조회할 수 있다
* 프로젝트별 통합 타임라인을 볼 수 있다
---
# 4. 대상 사용자
## 4.1 1차 타겟
* 임베디드/로보틱스/드론/캡스톤 프로젝트를 동시에 수행하는 개인 개발자
* 하드웨어 + 소프트웨어 + 문서 작업이 섞인 엔지니어
* Git 저장소와 실제 실험/제작 업무를 함께 관리해야 하는 사용자
## 4.2 핵심 사용자 페르소나
**박현수**
* 전자공학과 학생
* 임베디드, 드론, 성층권 풍선, GraphRAG 등 복수 프로젝트 진행
* GitHub/GitLab 혼용
* 실험, 배선, 센서 통합, 납땜, 펌웨어, 문서화를 동시에 수행
* 장기적으로 포트폴리오와 발표자료까지 정리해야 함

# 6. 핵심 사용 시나리오
## 6.1 프로젝트 생성
사용자는 새 프로젝트를 만들고, 설명/상태/저장소 연결 정보를 설정한다.
예:
* 성층권 풍선 데이터로거
* 안테나 트래커
* GraphRAG 캡스톤
## 6.2 실험 로그 작성
사용자는 실험의 목적, 조건, 결과, 문제점, 다음 액션을 기록한다.
예:
* BME688 온도 오프셋 테스트
* LoRa 송신거리 측정
* 저온 환경 센서 동작 확인
## 6.3 하드웨어 이슈 등록
사용자는 하드웨어 중심 문제를 구조적으로 기록한다.
예:
* GPS 콜드 스타트 지연
* SD 카드 SPI 충돌
* 저온 환경에서 센서 응답 이상
## 6.4 저장소 연동
사용자는 GitHub 또는 GitLab 저장소를 프로젝트에 연결하고 최근 커밋/이슈를 확인한다.
## 6.5 통합 흐름 파악
사용자는 프로젝트 상세 화면에서 아래를 한 번에 본다.
* 최근 실험 로그
* 열린 하드웨어 이슈
* 최근 커밋
* 최근 이슈/PR/MR

# 8. 비기능 요구사항
## 8.1 성능
* 주요 리스트 화면은 2초 이내 로딩
* 프로젝트 상세 조회는 3초 이내
* 첨부파일 업로드 시 진행 상태 표시
## 8.2 보안
* GitHub/GitLab 토큰은 서버에 암호화 저장
* HTTPS 사용
* Webhook secret 검증
* 최소 권한 원칙 적용
* NAS 관리자 계정과 앱 서비스 계정 분리
## 8.3 가용성
* Synology NAS 재부팅 후 컨테이너 자동 재시작
* DB 데이터 영속 저장
* 업로드 파일 영속 저장
## 8.4 확장성
* 추후 AI 요약, 포트폴리오 생성, 푸시 알림 기능 추가 가능해야 함
* GitHub/GitLab 외 다른 소스 확장 가능해야 함

# 9. 시스템 개요
## 9.1 아키텍처
* 프론트엔드: 모바일 앱(우선), 이후 웹 대시보드
* 백엔드: FastAPI
* DB: PostgreSQL
* 배포: Docker Compose on Synology NAS
* 저장소: NAS 공유 폴더
* 연동: GitHub API, GitLab API, Webhook
## 9.2 운영 환경
* Synology NAS
* DSM Container Manager
* Reverse Proxy + HTTPS
* Docker 기반 배포

# 14. 성공 지표
## MVP 성공 기준
* 3개 이상의 프로젝트를 앱 안에서 운영 가능
* 실험 로그 30건 이상 누적 가능
* 하드웨어 이슈를 구조적으로 재조회 가능
* GitHub/GitLab 최근 활동을 앱에서 확인 가능
* 프로젝트별 최근 상태를 1분 내 파악 가능
## 정성 지표
* 프로젝트 맥락 전환 시간이 줄어듦
* “다음에 뭘 해야 하지?”에 대한 혼란 감소
* 발표자료/포트폴리오 작성 시 정리 비용 감소

# 15. 리스크
## 15.1 기술 리스크
* GitHub/GitLab API 인증 처리 복잡성
* Synology NAS 외부 webhook 노출 보안
* 모바일 앱과 웹 대시보드 동시 개발 시 범위 증가
## 15.2 제품 리스크
* 기능이 너무 많아져 MVP가 무거워질 수 있음
* 일반 PM 앱처럼 흐르면 차별성이 사라짐
* 하드웨어 문맥 입력이 번거로우면 사용성이 떨어질 수 있음
## 15.3 대응
* MVP에서는 읽기 중심 연동으로 제한
* 입력 폼 단순화
* 실험 로그 + 하드웨어 이슈 중심으로 차별화
